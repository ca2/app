# acme Doxygen Homepage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate a useful `acme` documentation homepage and apply a durable ca2 visual theme while retaining Doxygen's wide, dense API interface and automatic light/dark behavior.

**Architecture:** A tracked Markdown file supplies the main-page content, and one additive stylesheet overrides Doxygen CSS variables and a small set of stable layout selectors. The existing Doxygen configuration references both inputs and enables native tree navigation; generated HTML remains disposable and is never edited.

**Tech Stack:** Doxygen 1.17, Doxygen Markdown and `@ref`, CSS custom properties, PowerShell verification, MSBuild, in-app browser responsive/theme verification.

---

## File Structure

- Create `acme/mainpage.md`: curated overview and stable links into generated API pages.
- Create `acme/doxygen-ca2.css`: ca2 typography, coordinated light/dark palettes, wide layout, compact navigation, and responsive rules.
- Modify `acme/doxygen.doxygen`: register the main page and stylesheet and enable Doxygen's native tree navigation.
- Do not modify `documentation/doxygen/**`: this directory is generated output.
